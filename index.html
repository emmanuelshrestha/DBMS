Database Management System

Data refers to raw facts, figures, or symbols that are processed to produce meaningful information. It can be in various formats such as text, numbers, audio, or images and is the backbone of any computational or digital system. Data becomes meaningful only when it is processed and analyzed in a structured way. In modern systems, data is crucial for decision-making, system operations, automation, and analytics.

1.1 Data Types

Data types define the type of data a variable can hold and how the computer interprets that data. Proper data typing ensures efficient data storage and accurate computations. Common data types include:

  • Integer: Stores whole numbers (e.g., 1, -45). Used for counting or indexing.
  • Float/Double: Stores decimal numbers (e.g., 3.14, -0.001). Used in scientific calculations.
  • Character: Represents a single alphabet, number, or symbol (e.g., 'A', '5').
  • String: A sequence of characters (e.g., "Hello World"). Used for names, sentences, etc.
  • Boolean: Represents true or false. Used in logic operations.
  • Date/Time: Holds date and time values for scheduling, timestamps, etc.

A database is an organized collection of structured information or data, typically stored electronically in a computer system. A database allows for efficient storage, retrieval, manipulation, and deletion of data.

2.1 Types of Databases

  • Relational Databases: Store data in tables with rows and columns. Data can be related using keys (e.g., MySQL, PostgreSQL).
  • NoSQL Databases: Non-tabular databases that store data in formats like documents, graphs, key-value pairs (e.g., MongoDB, Cassandra).
  • Object-Oriented Databases: Store data in the form of objects, similar to object-oriented programming.
  • Hierarchical Databases: Organize data in a tree-like structure. Each child has only one parent (e.g., IBM IMS).
  • Network Databases: More complex than hierarchical; records can have multiple parent and child relationships.

Domain-specific software or systems are specialized applications tailored to meet the unique requirements of particular industries, fields, or professional practices.

3.1 Database Management System (DBMS)

A DBMS is software that interacts with the database, users, and applications. It handles data definition, storage, retrieval, and security. Examples include Oracle, MySQL, and Microsoft SQL Server.

3.2 Accounting System

Accounting systems manage financial data. They handle invoices, payroll, tax computation, balance sheets, and general ledgers. Examples include Tally, QuickBooks, and Zoho Books.

3.3 Attendance System

These systems track attendance records of employees or students. Technologies like RFID, biometrics, QR codes, or online check-ins are commonly used.

The process of creating, designing, deploying, and maintaining software applications.

4.1 Database Design

Creating a detailed data model, defining tables, fields, relationships, indexes, and constraints. A good design avoids redundancy, ensures data integrity, and supports performance.

4.2 Database Administration

A database administrator (DBA) is responsible for installing, configuring, upgrading, monitoring, and securing the database system.

A server is a powerful computer system that provides services, data, and applications to other computers (clients) over a network. In a database environment, the server hosts the DBMS and processes client queries.

SQL is a standard and widely-used programming language designed specifically for managing and manipulating relational databases.

DDL (Data Definition Language)

  • CREATE: Creates new tables, views, or databases.
  • ALTER: Alters the structure of existing database objects.
  • DROP: Permanently deletes an object from the database.
  • TRUNCATE: Removes all records from a table, but retains its structure.

DML (Data Manipulation Language)

  • INSERT: Adds new rows into a table.
  • UPDATE: Modifies existing data based on specified criteria.
  • DELETE: Removes rows from a table based on a condition.

DQL (Data Query Language)

  • SELECT: Retrieves data from tables, often with WHERE, GROUP BY, etc.

DCL (Data Control Language)

  • GRANT: Gives user privileges.
  • REVOKE: Removes granted privileges.
  • INNER JOIN: Returns only records with matching values in both tables.
  • LEFT JOIN: All records from the left, matched ones from the right.
  • RIGHT JOIN: All records from the right, matched ones from the left.
  • FULL JOIN: All records when there is a match in either table.
  • SELF JOIN: Joins a table to itself.
  • CROSS JOIN: Returns Cartesian product of both tables.
  • Create: Adds new records (SQL: INSERT).
  • Read: Retrieves records (SQL: SELECT).
  • Update: Modifies records (SQL: UPDATE).
  • Delete: Removes records (SQL: DELETE).
  • Primary Key: Uniquely identifies each record.
  • Foreign Key: Links one table to another.
  • Composite Key: Combination of two or more columns.
  • Alternate Key: Candidate key not selected as primary.
  • Super Key: Any set of attributes that uniquely identifies a record.
  • NOT NULL: Ensures a column cannot have NULL values.
  • UNIQUE: Ensures all values are unique.
  • PRIMARY KEY: Combines NOT NULL and UNIQUE.
  • FOREIGN KEY: Enforces referential integrity.
  • CHECK: Ensures all values satisfy a specific condition.
  • DEFAULT: Sets a default value for a column.
  • Insertion Anomaly: Trouble inserting data due to missing other fields.
  • Update Anomaly: Inconsistencies from updating data in multiple places.
  • Deletion Anomaly: Losing useful data when deleting unrelated info.
  • 1NF: Eliminate repeating groups.
  • 2NF: Remove partial dependencies.
  • 3NF: Remove transitive dependencies.
  • BCNF: Higher version of 3NF; every determinant is a candidate key.
  • Authentication: Verifying identity.
  • Authorization: Granting access rights.
  • Encryption: Protecting data in transit and at rest.
  • Backup and Recovery: Prevent data loss.
  • Audit Trails: Track who did what and when.
